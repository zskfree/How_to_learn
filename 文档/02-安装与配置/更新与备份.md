# 更新与备份

Clash Verge、配置文件和文档中的平台入口都可能变化，需要定期维护。

## 建议备份内容

- Clash Verge 配置文件。
- 规则配置。
- 本项目的 `配置/工具与平台地址.yml`。
- 重要截图和教程素材。

## 更新顺序

1. 先备份当前可用配置。
2. 再更新 Clash Verge 或配置文件。
3. 更新后执行连通性检查。
4. 如果平台入口变化，更新 `配置/工具与平台地址.yml`。
5. 如教程步骤受影响，再更新对应 Markdown 文档。

## 不建议做的事

- 不要在没有备份的情况下覆盖可用配置。
- 不要把敏感配置提交或分享给他人。
- 不要因为单个平台打不开就立即大范围修改系统网络设置。
