# 平台地址更新流程

平台地址、推荐等级和状态统一维护在 `配置/工具与平台地址.yml`。

## 更新频率

- 每月检查：主推 AI 平台、核心科研平台、常用搜索型 AI 平台。
- 每季度检查：浏览器、文献管理工具、出版社数据库和低频工具。
- 临时检查：读者反馈无法访问、平台改名、入口迁移、功能大幅调整。

## 检查优先级

1. 主推 AI 平台：`AI平台.chatgpt`、`AI平台.claude`、`AI平台.gemini`。
2. 核心科研平台：`科研平台.google_scholar`、`科研平台.semantic_scholar`、`科研平台.arxiv`、`科研平台.pubmed`。
3. 搜索和文献 AI：`AI平台.perplexity`、`AI平台.consensus`、`AI平台.elicit`。
4. 国内 AI 平台：`AI平台.kimi`、`AI平台.tongyi`、`AI平台.deepseek`、`AI平台.doubao` 等。
5. 其他工具和数据库。

## 更新步骤

1. 打开 `配置/工具与平台地址.yml`。
2. 检查条目的 `官网` 是否仍然可用。
3. 检查 `用途`、`推荐等级`、`访问方式` 是否需要调整。
4. 更新 `上次检查`。
5. 更新 `状态`，例如 `可用`、`待验证`、`入口变化`、`暂不可用`。
6. 如果文档中的说明受影响，再更新对应 Markdown。
7. 如有必要，在 `报告` 目录记录本次检查结果。

## 新增平台规则

新增平台时，必须包含：

- 名称
- 厂商
- 地区
- 推荐等级
- 官网
- 访问方式
- 用途
- 适用文档
- 上次检查
- 检查周期
- 状态

## 不写入的内容

- 账号和密码。
- API Key。
- 订阅链接。
- 节点信息。
- 机构内部访问地址。
- 未授权资源入口。
