# 如何通过科学工具辅助科研

本项目是一个面向科研学习者的中文文档库，目标是帮助读者理解如何合规、稳定地使用网络工具访问学术资源，并使用国内外 AI 平台辅助论文检索、阅读、总结、写作和代码分析。

## 阅读路径

1. 先读 `文档/00-项目说明/项目定位.md` 和 `文档/00-项目说明/合规与使用边界.md`，明确项目用途和边界。
2. 再读 `文档/01-网络基础/基础概念.md`，理解代理、规则、节点、连通性检查等基础概念。
3. 按 `文档/02-安装与配置/Windows安装Clash-Verge.md` 安装并完成基础配置。
4. 进入 `文档/03-科研资源访问`，学习谷歌学术、论文数据库和文献检索工作流。
5. 进入 `文档/04-AI大模型辅助科研`，优先学习 GPT、Claude、Gemini 的科研使用方式。
6. 遇到问题时查 `文档/05-常见问题与排错`。

## 统一地址配置

工具官网、科研平台、AI 平台地址统一维护在：

- `配置/工具与平台地址.yml`

文档正文尽量只引用平台 ID，例如：

- `AI平台.chatgpt`
- `AI平台.claude`
- `AI平台.gemini`
- `科研平台.google_scholar`
- `工具.clash_verge`

后续更新平台入口、状态、检查周期时，优先修改 YAML 配置文件，再按需同步推荐工具索引。

首页推荐平台链接由 `npm run build:pages` 从 YAML 配置生成；提交前可运行 `npm run check` 检查本地链接、图片引用、YAML 可读性和需要人工核验的动态链接。

定期更新流程见：

- `文档/99-资料与模板/平台地址更新流程.md`

## 目录说明

- `配置`：统一维护工具、科研平台、AI 平台地址。
- `文档`：主教程文档。
- `资源`：教程截图、流程图等素材。
- `安装包`：可临时存放本地安装包；二进制安装包不提交到 Git，优先使用官方 Releases 下载。
- `案例`：完整科研场景案例。
- `报告`：后续验证记录、检查报告和更新记录。

## 使用边界

本项目仅面向科研学习、文献检索和 AI 辅助研究。请遵守所在地法律法规、学校或机构网络政策、数据库订阅规则和平台服务条款。不要将本文档用于绕过版权、入侵系统、滥用平台或违反机构规定的行为。
